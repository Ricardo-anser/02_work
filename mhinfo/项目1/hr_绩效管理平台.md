# HR 绩效管理平台

## 项目定位

企业内部人力资源与绩效管理系统，覆盖绩效沟通、绩效面谈、权限控制、流程适配和批量处理等场景。我的价值重点在于把复杂流程型业务真正落到系统里，而不是只做单点页面开发。

## 技术关键词

- Java 8
- Spring Boot
- Spring Cloud
- Nacos
- MySQL
- Vue 2
- Element UI
- 流程引擎
- 权限控制
- Excel 导入导出

## 我负责的内容

- 负责绩效沟通、绩效面谈相关需求开发和问题修复
- 联动前后端处理面谈人查询、领导选择、审批人传值等关键链路问题
- 补齐批量导入、批量导出、批量提交、失败反馈等业务操作能力
- 支持流程引擎和权限体系适配，沉淀环境说明、权限说明和交接资料

## 可写进简历的亮点

- 围绕绩效沟通和绩效面谈链路做前后端联调，提升流程完整度和业务可用性
- 优化面谈人查询、组织树映射和监控页人员展示问题，兼顾性能与准确性
- 补齐批量导入导出、预览和错误回显能力，降低业务继续依赖线下处理的概率
- 打通前端入口、后端接口权限与流程调用方式，减少“页面有功能但业务走不通”的问题

## 简历短版

负责 HR 绩效管理模块的前后端联调与功能优化，完成绩效沟通/绩效面谈场景下的面谈人查询优化、领导选择、批量导入导出与提交流程改造，并推动流程权限适配和交接文档沉淀。

## 面试展开点

- 为什么“面谈人查询慢”本质上可能是组织树和流程关系问题，而不是单纯接口慢
- 为什么绩效类系统要重视批量导入导出和失败回显
- 权限、流程、页面三者没有打通时，业务侧会感知成什么问题
