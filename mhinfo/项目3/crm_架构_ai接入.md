# 供销服一体化 CRM 平台

## 项目定位

复杂存量 CRM 业务系统的架构梳理、业务建模与 AI 接入分析项目。我的价值重点在于把多系统、多业务域的复杂结构整理成可理解、可汇报、可继续演进的系统认知。

## 技术关键词

- Vue 2
- Element UI
- Spring Boot
- MySQL
- Node.js
- Express
- Sequelize
- Redis
- Elasticsearch
- Flowable
- OnlyOffice
- Draw.io

## 我负责的内容

- 梳理 `crm-ui`、`crm-back`、`wms-node-service` 三层系统边界和交互关系
- 对客户、商机、订单、合同等核心链路做业务流程和函数级拆解
- 结合测试库、SQL 和源码证据梳理数据库关系，输出业务表与系统表关系图
- 评估现有知识库问答和聊天能力，设计审批、库存、对账等场景的 AI Agent 接入路径

## 可写进简历的亮点

- 把双后端、多业务域的复杂存量系统整理成结构化架构图、业务图和说明文档
- 在无真实外键约束的情况下，通过 SQL join 证据和脚本分析还原业务关系
- 基于现有 Node 编排层能力提出低侵入、可治理的 AI Agent 接入方案
- 为后续汇报、接手和系统演进提供统一认知底座

## 简历短版

负责复杂 CRM 存量系统的架构与业务梳理，输出三层系统架构、核心业务流程和数据库关系图，并结合现有 Node 编排层提出审批、库存、对账等场景的 AI Agent 接入方案。

## 面试展开点

- 如何快速读懂一个双后端、多业务域的复杂系统
- 在没有外键约束时，如何还原真实业务关系
- AI 接入为什么要先看边界、治理和 ROI，而不是只谈模型能力
