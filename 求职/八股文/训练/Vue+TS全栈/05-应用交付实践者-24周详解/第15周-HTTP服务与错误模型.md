# 第15周 HTTP 服务与错误模型

## 这周为什么重要

随着接口越来越多，如果没有统一错误出口和中间件结构，联调会越来越痛。

## 先讲明白原理

- 每个接口都自己处理所有事情，代码会迅速重复。
- 中间件的价值是把鉴权、日志、错误处理、参数校验这些共性动作集中管理。
- 错误模型稳定，前端页面才能稳定。

## 最小例子

```ts
return {
  ok: false,
  code: "UNAUTHORIZED",
  message: "请先登录",
};
```

这个结构看起来普通，但它已经比“有时返回 message，有时返回 error”强很多。

## 放进真实项目怎么理解

- Vue 页面可以更稳定地写错误提示。
- TS 可以让错误码成为明确枚举，而不是字符串乱飞。
- Node 可以通过统一中间件把共性逻辑收口。

## 实际应用场景

- 接口鉴权失败。
- 参数校验失败。
- 业务冲突，比如重复创建。

## 容易踩的坑

- 全部错误都返回 500。
- 某些接口绕过中间件单独玩一套结构。
- 前端为了兼容多套错误结构，写一堆 if/else。
