# 第23周 部署与运行手册

## 这周为什么重要

项目如果只能在你电脑上跑，那它还不算真正交付。第 23 周就是把“个人作品”推进到“可运行系统”。

## 先讲明白原理

- 部署关注的是上线动作。
- 运行手册关注的是上线后如何验证、如何排障、如何回滚。
- 你真正交付给团队或面试官的，不只是代码，还有能让别人接手的说明。

## 最小例子

一个最小 runbook 至少要回答：

1. 环境变量有哪些
2. 数据库和缓存如何准备
3. 启动命令在哪个目录执行
4. 用什么方式验证服务正常
5. 出问题先查哪里

## 放进真实项目怎么理解

- Vue 产物部署关注静态资源和环境变量。
- Node 服务部署关注依赖服务和启动顺序。
- 数据迁移、缓存预热、日志目录都可能是上线风险点。

## 实际应用场景

- 本地模拟上线。
- 测试环境部署。
- 面试演示时快速启动项目。

## 容易踩的坑

- README 只有一句“npm run dev”。
- 不写环境变量说明。
- 没有健康检查和回滚思路。

- 明确命令执行上下文。
- 解释为什么“可交接性”也是交付质量的一部分。
