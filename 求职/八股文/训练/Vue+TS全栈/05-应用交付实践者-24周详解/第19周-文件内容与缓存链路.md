# 第19周 文件、内容与缓存链路

## 这周为什么重要

这周开始，项目要从“数据管理”进入“内容系统”层面，复杂度会明显上升。

## 先讲明白原理

- 内容系统通常要同时处理文本、文件、缓存、预览、渲染。
- 文件链路和内容链路会在页面展示阶段交汇。
- 一旦涉及上传和缓存，就必须开始思考校验与失效策略。

## 最小例子

```ts
type UploadResult = {
  url: string;
  md5: string;
};
```

这类结构说明的不是“上传成功了”，而是“上传后系统如何继续追踪和校验这个资源”。

## 放进真实项目怎么理解

- 前端编辑器或表单要能引用上传结果。
- 后端要存储内容、处理附件、维护缓存。
- SSR 内容页还要考虑首屏展示与缓存更新。

## 实际应用场景

- 文章编辑与预览。
- 图片上传与引用。
- 内容详情页缓存。

## 容易踩的坑

- 上传成功了却没有回显策略。
- 内容更新了但缓存没失效。
- 只做了 happy path，没有坏文件和空内容处理。

## 本周交付

- 打通内容创建、上传、渲染或预览最小链路。
- 为上传结果加上基本校验信息。
- 解释为什么内容系统一旦做起来，会比后台表单复杂很多。
