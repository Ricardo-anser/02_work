# 第06周 Vue 渲染与更新

## 这周为什么重要

项目开始长页面之后，问题不再是“能不能显示出来”，而是“为什么改一点东西，整个页面都跟着动”。

## 先讲明白原理

- 渲染边界决定哪些组件该更新，哪些不该更新。
- 组件拆得合理，更新路径就清晰。
- 状态边界混乱，哪怕框架再聪明，也会出现大量无效渲染。

## 最小例子

```vue
<TodoList :items="items" />
<TodoSummary :count="items.length" />
```

关键点不是两行组件代码，而是你有没有想清楚：谁真正依赖 `items`，谁只依赖 `count`。

## 放进真实项目怎么理解

- 列表项组件要独立。
- 详情区和列表区尽量解耦。
- 同一页面的多个区域不应该因为一个输入框就全重刷。

## 实际应用场景

- 列表筛选。
- 表格行内编辑。
- 详情抽屉与主列表联动。

## 容易踩的坑

- key 写错。
- 所有逻辑和状态塞进一个大页面组件。
- 一改筛选条件，整个页面所有区域都跟着抖。
