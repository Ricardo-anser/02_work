# 第12周 类型驱动工具

## 这周为什么重要

到了这里，你已经不只是在“把类型写对”，而是要让类型反过来服务项目效率。

## 先讲明白原理

- 好工具不是大而全，而是正好解决你当前最痛的重复劳动。
- 类型驱动工具最常见的价值，是减少重复结构、减少猜字段、减少手写映射。
- 对应用交付来说，工具是否值得保留，取决于它是否真实节省时间。

## 最小例子

```ts
function createApiPath<T extends string>(path: T) {
  return `/api/${path}`;
}
```

这类工具虽小，但一旦与更完整的类型约束结合，就能帮你稳定命名与调用方式。

## 放进真实项目怎么理解

- typed client。
- 表单 schema 辅助器。
- 统一错误码映射工具。

## 实际应用场景

- 请求封装。
- 表单提交。
- 埋点事件命名。

## 容易踩的坑

- 工具本身太抽象，团队没人敢动。
- 只做 demo，不接入真实项目。
- 一开始就想做全栈代码生成器，最后烂尾。
