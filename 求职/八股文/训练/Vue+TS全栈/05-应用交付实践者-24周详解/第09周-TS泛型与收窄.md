# 第09周 TS 泛型与收窄

## 这周为什么重要

项目刚开始时 `any` 还能蒙混过关，到了第二阶段它就会开始制造联调事故。

## 先讲明白原理

- 泛型能让你写通用逻辑时不丢失信息。
- 收窄能让你处理复杂分支时更安全。
- 对应用交付来说，这两点最直接的收益就是：代码可复用，页面不乱猜。

## 最小例子

```ts
function request<T>(url: string): Promise<T> {
  return fetch(url).then((res) => res.json());
}
```

重点不是这个函数够不够严谨，而是：调用方终于能得到“这次请求到底回什么”的精确信息。

## 放进真实项目怎么理解

- 列表请求、详情请求、分页容器都适合用泛型。
- 错误对象解析、接口判别结果都依赖收窄。
- 越到中后期，越要把模糊类型清掉。

## 实际应用场景

- 通用请求封装。
- 表格列定义。
- 错误处理工具。

## 容易踩的坑

- 泛型只是包一层，里面还是 any。
- 错误对象全靠断言。
- 项目越大，隐式 any 越多。
