# 第16周 鉴权、数据库与缓存

## 这周为什么重要

从这周开始，项目从“演示级”进入“接近真实业务级”。因为用户身份、数据持久化、缓存策略会同时出现。

## 先讲明白原理

- 鉴权解决“能不能访问”。
- 数据库解决“业务事实放在哪里”。
- 缓存解决“哪些结果值得用空间换时间”。
- 交付真实系统时，这三层缺一不可。

## 最小例子

```ts
type CurrentUser = {
  id: string;
  role: "admin" | "editor";
};
```

这个结构背后其实已经包含了权限系统的最小骨架：身份、角色、访问边界。

## 放进真实项目怎么理解

- 前端需要登录态和权限态。
- 后端需要 token 校验、数据库查询、缓存命中策略。
- shared types 需要稳定表达用户与权限相关结构。

## 实际应用场景

- 后台登录。
- 用户资料读取。
- 内容列表缓存。

## 容易踩的坑

- 只做登录，不做权限控制。
- 把数据库结果原样返回给前端。
- 加了缓存却没有失效策略。
