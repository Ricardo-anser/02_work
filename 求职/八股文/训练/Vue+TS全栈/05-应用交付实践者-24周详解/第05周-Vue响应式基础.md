# 第05周 Vue 响应式基础

## 这周为什么重要

应用交付路线不要求你手撕全部响应式源码，但你必须会把页面状态组织得稳定，不然业务一复杂就会炸。

## 先讲明白原理

- 页面本质上是状态的可视化。
- 状态组织不好，页面就会出现：值改了 UI 不变、UI 变了但状态不同步、多个地方改同一状态互相打架。
- Vue3 响应式帮你自动追踪变化，但前提是你得先建对状态模型。

## 最小例子

```ts
const form = reactive({
  title: "",
  loading: false,
  error: "",
});
```

看起来只是 3 个字段，但它已经让你可以清楚地表达：用户输入、请求状态、失败反馈。

## 放进真实项目怎么理解

- 表单页要明确局部状态。
- 列表页要明确数据状态。
- 页面间共享状态不能和局部输入态混淆。

## 实际应用场景

- 登录表单。
- 创建文章表单。
- 列表筛选和分页。

## 容易踩的坑

- 所有状态都放组件顶层。
- 派生值手动同步。
- 把请求副作用和纯状态写在一起。
