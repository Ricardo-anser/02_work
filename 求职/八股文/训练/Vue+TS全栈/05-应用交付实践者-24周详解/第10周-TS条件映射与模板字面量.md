# 第10周 TS 条件、映射与模板字面量

## 这周为什么重要

项目里一旦出现大量重复字段、重复事件名、重复错误结构，你就会很想把它们自动化。

## 先讲明白原理

- 条件类型帮你表达“这个条件下返回什么类型”。
- 映射类型帮你表达“同一批字段做同一种加工”。
- 模板字面量帮你表达“字符串命名也有结构”。

## 最小例子

```ts
type FieldErrors<T> = {
  [K in keyof T]?: string;
};
```

一个表单字段变了，错误结构也跟着变，这就是它对真实交付的价值。

## 放进真实项目怎么理解

- 表单错误映射。
- 事件命名规范。
- 统一响应包装结构。

## 实际应用场景

- 登录表单。
- 内容编辑表单。
- 用户操作事件埋点。

## 容易踩的坑

- 为了炫技把类型写成谜语。
- 自动化掉重复，却让新人完全看不懂。
- 把本来简单的对象硬拗成复杂类型体操。
