# 第11周 类型共享与接口对齐

## 这周为什么重要

到了这一周，你应该正式把“前端自己写一份类型，后端自己写一份类型”的低效模式结束掉。

## 先讲明白原理

- 共享类型解决的是跨层一致性，不是方便复制粘贴。
- 真正值得共享的是 DTO、错误结构、分页容器、认证结果这类传输协议。
- 内部数据库实现细节通常不该直接暴露给前端。

## 最小例子

```ts
export type PageResult<T> = {
  list: T[];
  total: number;
};
```

这种共享容器非常适合在真实项目里先落地，因为它清晰、复用率高、争议少。

## 放进真实项目怎么理解

- 前端分页表格直接消费共享类型。
- 后端接口统一按共享结构返回。
- 类型一改，两边同时感知，能尽早发现破坏。

## 实际应用场景

- 用户列表。
- 内容列表。
- 搜索结果。

## 容易踩的坑

- 共享 ORM 模型。
- 共享路径乱、循环依赖多。
- 共享类型文件越堆越大，没有分层。

- 让 1 条真实业务链完全走共享契约。
- 解释为什么“共享契约”是多人协作效率杠杆。
