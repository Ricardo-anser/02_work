# 第20周 系统集成里程碑

## 这周为什么重要

这是应用路线最关键的收束周之一。你要把前面分散的功能，真正装成一个像样的系统。

## 先讲明白原理

- 集成不等于拼接页面。
- 集成真正要处理的是：依赖顺序、状态流向、鉴权边界、缓存更新、页面可见性。
- 交付一个系统，远比做多个孤立功能更考验设计能力。

## 最小例子

一条“登录 -> 发布内容 -> 访问内容详情”的链路至少会跨过：

1. 登录状态
2. 路由守卫
3. 接口契约
4. 数据库存储
5. 内容渲染
6. 缓存命中/失效

## 放进真实项目怎么理解

- Vue 页面负责用户路径连续性。
- TS 负责链路结构不漂移。
- Node 负责服务编排与状态一致性。

## 实际应用场景

- 博客/CMS。
- 内容后台。
- 带登录态的知识库系统。

## 容易踩的坑

- 模块都在，但主链路跑不通。
- 某一步成功后下一步状态没接上。
- 页面和接口都“差不多”，但组合起来就断。

## 本周交付

- 交付一个博客/CMS 集成版主链路。
- 记录最脆弱的 3 个系统断点。
- 解释为什么“系统思维”是从这周开始真正形成的。
