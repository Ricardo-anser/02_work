# 第08周 Vue 阶段里程碑

## 这周为什么重要

前面几周你把页面、状态、组件、store 都摸到了，这一周要把它们收束成一个可用的后台壳。

## 先讲明白原理

- 管理台壳不是业务系统的“皮”，它其实是之后所有业务页面的容器。
- 路由、登录态、布局、页面状态组织方式，都会在这里定型。
- 所以这周做得稳，后面 16 周会轻松很多。

## 最小例子

后台壳至少包含：

1. 登录页
2. 路由入口
3. 顶部或侧边布局
4. 一个受保护页面

## 放进真实项目怎么理解

- Vue Router 决定页面切换。
- Pinia 决定登录态和用户态。
- 页面布局决定后续业务扩展的舒适度。

## 实际应用场景

- 内容管理后台。
- 用户管理后台。
- 内部运营系统。

## 容易踩的坑

- 一开始没做守卫，后面补很痛。
- 所有业务都塞进首页组件。
- 布局和业务逻辑严重耦合。

## 本周交付

- 交付一个带登录态的最小管理台壳。
- 保证未登录不能进受保护页面。
- 解释为什么“后台壳”其实是后续业务扩展的地基。
