# 第14周 事件循环、Stream 与 Buffer

## 这周为什么重要

一旦你开始做上传、导出、日志处理，就不能再把所有东西都当成 JSON 请求看了。

## 先讲明白原理

- 事件循环影响请求吞吐和阻塞风险。
- Buffer 让你能处理二进制内容。
- Stream 让你能边收边处理，而不是一次性把整个文件吃进内存。

## 最小例子

```js
req.on("data", (chunk) => {
  console.log("收到字节块", chunk.length);
});
```

这段代码最重要的意义是：文件和大数据不是一次给你的，而是一块块流进来的。

## 放进真实项目怎么理解

- 头像上传、附件上传、导出报表都离不开流。
- 前端只看到“上传进度”，后端看到的是块状数据与写入策略。
- 这周会直接影响你做文件链路时会不会把服务内存打爆。

## 实际应用场景

- 文件上传。
- 报表下载。
- 大日志分析。

## 容易踩的坑

- 直接整文件读进内存。
- 以为异步就不会阻塞。
- 忽略上传失败后的残留文件清理。
