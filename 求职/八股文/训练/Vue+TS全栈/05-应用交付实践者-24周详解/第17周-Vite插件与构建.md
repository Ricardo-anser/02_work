# 第17周 Vite 插件与构建

## 这周为什么重要

到了这个阶段，前端已经不只是写页面，还要考虑产物、环境注入、构建稳定性和资源切分。

## 先讲明白原理

- 构建阶段是在帮你把源码变成适合浏览器和部署环境消费的产物。
- 插件系统让你可以把业务需求前移到构建期解决。
- 越早在构建期解决，运行时负担通常越低。

## 最小例子

```ts
define: {
  __APP_VERSION__: JSON.stringify("1.0.0")
}
```

这个注入看似简单，但它已经说明：有些信息适合在构建时决定，不必拖到运行时。

## 放进真实项目怎么理解

- 资源懒加载。
- 环境变量注入。
- 构建期生成路由、内容索引或版本信息。

## 实际应用场景

- 后台管理系统构建优化。
- 内容站静态生成辅助。
- 打包体积和首屏体验改进。

## 容易踩的坑

- 本地 dev 能跑，build 才发现问题。
- 环境变量用法不统一。
- 构建配置越加越乱，没有业务目的。

- 检查并优化 1 处懒加载或依赖引入方式。
- 解释为什么懂构建能直接提升交付可靠性。
