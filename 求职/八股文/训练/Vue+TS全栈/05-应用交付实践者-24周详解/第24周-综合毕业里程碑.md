# 第24周 综合毕业里程碑

## 这周为什么重要

最后一周不是再补零散知识，而是让你真正拿出一个“可以讲、可以演示、可以上线说明、可以复盘”的全栈作品。

## 先讲明白原理

- 应用交付的最终标准不是“功能越多越好”，而是“核心链路完整、边界清晰、失败可解释、部署可说明”。
- 一个能被讲清楚的项目，比一个堆了很多功能却说不明白的项目强得多。

## 最小例子

毕业作品至少要能回答：

1. 用户怎么登录
2. 业务数据怎么流动
3. 页面为什么这样组织
4. 接口为何这样设计
5. 出错时系统怎么处理

## 放进真实项目怎么理解

- Vue 页面负责交互体验。
- TS 负责契约稳定。
- Node 负责运行时能力。
- 部署与观测负责把项目拉近“真实系统”。

## 实际应用场景

- 博客/CMS 成品。
- 后台管理系统成品。
- 用于面试展示的完整全栈作品。

## 容易踩的坑

- 只准备 happy path 演示。
- 项目能跑，但讲不明白为什么这样分层。
- 文档缺失，别人根本起不来项目。

## 本周交付

- 完整演示项目主链路。
- 准备 1 份讲解稿和 1 份失败案例说明。
- 回答自己：这个项目最能代表我哪种能力。
