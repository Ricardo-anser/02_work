# 第18周 SSR、CSR 与 Hydration

## 这周为什么重要

当你从后台系统走向内容系统、博客系统、营销页面时，SSR 和 Hydration 就会变成非常实际的话题。

## 先讲明白原理

- CSR 适合交互重、首屏可稍慢的系统。
- SSR 更适合首屏内容可见性和 SEO 更重要的页面。
- Hydration 决定客户端怎样接手服务端已经渲染好的结果。

## 最小例子

如果页面模板里直接用了：

```js
const now = Date.now();
```

那服务端和客户端极有可能第一次渲染结果不同，这就是 SSR 场景下的实际风险。

## 放进真实项目怎么理解

- 后台管理系统通常不一定要 SSR。
- 内容详情页、博客页往往值得 SSR。
- 真正的判断标准不是“流不流行”，而是“业务页面需要什么”。

## 实际应用场景

- 博客详情页。
- 帮助中心文章页。
- 营销落地页。

## 容易踩的坑

- 为了跟风上 SSR。
- 忽略 hydration mismatch。
- 服务端和客户端取数方式完全割裂。
