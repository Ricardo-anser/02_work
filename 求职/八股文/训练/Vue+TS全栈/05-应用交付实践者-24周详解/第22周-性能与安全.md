# 第22周 性能与安全

## 这周为什么重要

越接近真实交付，越不能只盯着功能。性能和安全不是锦上添花，而是能不能放心上线的底线。

## 先讲明白原理

- 性能优化不是无脑缓存，而是先知道瓶颈在哪。
- 安全设计不是只防注入，而是从输入、权限、输出三端同时看。
- 很多“线上事故”表面是 bug，本质是性能或安全边界没立住。

## 最小例子

```ts
if (!currentUser) {
  throw new Error("UNAUTHORIZED");
}
```

最小权限检查看上去简单，却能提醒你：没有显式边界，系统默认就是不安全的。

## 放进真实项目怎么理解

- Vue 侧要关注首屏、懒加载、无效更新。
- Node 侧要关注慢查询、阻塞任务、输入校验。
- 内容系统要额外关注上传和渲染的安全边界。

## 实际应用场景

- 登录权限控制。
- 内容渲染。
- 文件上传。

## 容易踩的坑

- 只看功能是否完成，不看资源体积。
- 只做登录，不做资源级权限。
- 上传接口信任前端传来的类型信息。
