# 应用交付实践者：Vue3 + TypeScript + Node.js 24 周学习指南

## 00. 使用说明

- 路线目标：在 24 周内完成“能交付、能联调、能部署、能观测”的实战闭环。
- 训练原则：每一周都要落到可运行功能，每一阶段都要形成可验收系统。
- 非妥协约束：
  - 鉴权必须出现 JWT 语义。
  - 数据层必须出现 PostgreSQL 语义。
  - 缓存层必须出现 Redis 语义。
  - SSR/CSR 差异必须在内容系统阶段被显式比较。
  - 监控与错误上报必须在后期阶段落位。

## 01. 前置知识

### 必须掌握

- JavaScript 基础语法与 DOM。
- 使用包管理器安装依赖、运行脚本。
- HTTP 基础请求/响应概念。

### 训练中补齐

- TypeScript 的基础注解与泛型直觉。
- SQL 的 CRUD 与关系模型。
- Git 的最常见操作。

### 暂时忽略

- 大型组件库深度定制。
- 微服务拆分与服务网格。
- 复杂消息队列与分布式事务。
