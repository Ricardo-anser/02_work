# 第23周 部署与运行手册

## 本周你到底在学什么

你这周要意识到：能在自己电脑上跑，不等于系统真的“可运行”。

## 先把原理讲透

- 部署关注的是“怎么把代码和依赖放到目标环境里跑起来”。
- 运行手册关注的是“跑起来之后，怎么验证、怎么排障、怎么回滚”。
- 真正成熟的系统，除了代码，还要有操作说明。

## 最小例子

一个最小运行说明至少应该回答：

1. 需要哪些环境变量
2. 先启动哪些依赖
3. 用什么命令启动应用
4. 如何验证服务可用
5. 出问题先看哪里

## 放到 Vue3 / TS / Node.js 里

- Vue 前端通常涉及构建产物、静态资源托管、环境变量注入。
- TS 构建配置决定上线产物是否稳定。
- Node 服务通常还牵涉数据库、缓存、迁移脚本、日志目录。

## 实际应用场景

- 新同事接手项目。
- 测试环境回归部署。
- 生产事故后的快速恢复。

## 常见误区

- 误区：写个 `npm run start` 就算部署说明。
  - 真正困难的是依赖顺序、环境变量和验证动作。
- 误区：回滚就是重新发旧代码。
  - 数据结构变化、缓存状态、资源版本都可能影响回滚。

## 本周实践

- 给你的项目写一份最小 runbook。
- 明确健康检查和就绪检查分别看什么。
- 列出 3 个“开发环境感觉不到，上线才会暴露”的问题。

## 学完你应该能说清

- 部署说明和运行手册的区别。
- 为什么系统交付一定包含操作知识。
- 为什么“没有 runbook 的项目”在团队里风险很高。
