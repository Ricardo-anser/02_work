# 第09周 TS 泛型与收窄

## 本周你到底在学什么

这周不是学高级语法炫技，而是学“怎么让函数在复用时仍然保留信息，而不是一路退化成 any”。

## 先把原理讲透

- 泛型解决的是“逻辑相同，但处理的数据类型不同”。
- 收窄解决的是“变量起点很宽，但在分支里可以越来越具体”。
- 这两件事加起来，才让 TS 真正从注释工具变成推理工具。

## 最小例子

```ts
function first<T>(list: T[]): T | undefined {
  return list[0];
}

const name = first(["Ada", "Grace"]);
```

关键问题是：为什么这里 `name` 会被推断成 `string | undefined`，而不是模糊的 unknown。

## 放到 Vue3 / TS / Node.js 里

- Vue3 里组件 props、hooks 返回值、表单工具函数都依赖泛型。
- Node.js 里请求层、分页容器、服务结果模型都适合用泛型抽象。
- 收窄在错误处理、判别联合、第三方返回值处理里特别常见。

## 实际应用场景

- 通用分页工具。
- 通用表格列定义。
- 统一 API 结果容器。

## 常见误区

- 误区：函数能跑就行，泛型以后再说。
  - 后面会发现复用时信息全丢了。
- 误区：类型收窄只是写 `typeof`。
  - 实际上判别联合、自定义守卫都属于收窄。

## 本周实践

- 自己写 3 个泛型函数：`first`、`pick`、`wrapResult`。
- 写一个可辨识联合错误模型并在分支里收窄。
- 找出现有项目里 2 处“本该用泛型却用了 any”的地方。

## 学完你应该能说清

- 泛型为什么是“延迟决定类型”的机制。
- 收窄为什么能让分支内代码更安全。
- 为什么泛型是项目可维护性的投资，不是语法表演。
