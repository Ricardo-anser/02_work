# 第16周 鉴权、数据库与缓存

## 本周你到底在学什么

这周要理解三个经常被混着说的东西：用户身份、持久数据、临时加速层。

## 先把原理讲透

- 鉴权回答的是“你是谁，你能做什么”。
- 数据库回答的是“业务事实永久存在哪里”。
- 缓存回答的是“哪些结果值得临时保留，换速度”。
- 这三者协同得好，系统又快又稳；协同不好，就会又乱又难查。

## 最小例子

```ts
type Session = {
  userId: string;
  token: string;
  expiresAt: number;
};
```

只看这一个结构，你就可以思考三层问题：身份凭证怎么来，用户信息查哪，过期信息是否缓存。

## 放到 Vue3 / TS / Node.js 里

- Vue 页面只看见“是否登录、当前用户是谁、有没有权限”。
- TS 帮你把 token、user、permission 这些契约说清。
- Node 则负责 JWT 校验、DB 查询、Redis 缓存命中与失效。

## 实际应用场景

- 登录成功后返回 token。
- 请求个人资料时先看缓存再查库。
- 多租户系统用数据库隔离保证权限边界。

## 常见误区

- 误区：JWT 就是登录系统。
  - JWT 只是身份凭证表达方式，不等于完整鉴权体系。
- 误区：缓存就是把数据复制一份，越多越快。
  - 没有失效策略的缓存很快会变成脏数据源。

## 本周实践

- 画一张“登录 -> 鉴权 -> 查用户 -> 返回页面”的链路图。
- 分清哪些数据应该进数据库，哪些更适合缓存。
- 用通俗话解释“为什么缓存不是银弹”。

## 学完你应该能说清

- 鉴权、数据库、缓存三者分别解决什么问题。
- JWT 为什么不能单独承担权限设计。
- 为什么缓存命中率高不代表系统设计就健康。
