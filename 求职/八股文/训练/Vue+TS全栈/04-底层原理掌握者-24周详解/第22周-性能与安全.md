# 第22周 性能与安全

## 本周你到底在学什么

这一周的重点不是背优化 checklist，而是把“慢”和“危险”都还原成边界失控问题。

## 先把原理讲透

- 性能问题常见于：重复计算、无效更新、阻塞 I/O、过大资源、错误缓存。
- 安全问题常见于：输入不可信、权限边界不清、敏感信息泄露、输出未过滤。
- 这两类问题看似不同，但都和“约束不明确”强相关。

## 最小例子

```js
for (let i = 0; i < 1e9; i++) {}
```

这一行代码会阻塞事件循环。它提醒你：不是所有“能运行的代码”都适合放在服务请求路径上。

## 放到 Vue3 / TS / Node.js 里

- Vue 里常见的是无效渲染、过大 bundle、首屏阻塞。
- TS 不能直接提升性能，但能通过更清晰的边界减少错误设计。
- Node 里常见的是阻塞事件循环、慢查询、错误缓存、安全校验不足。

## 实际应用场景

- 表单校验。
- 富文本或 Markdown 内容渲染。
- 上传接口、登录接口、搜索接口。

## 常见误区

- 误区：性能优化就是做缓存。
  - 错误缓存会让问题更难发现。
- 误区：安全就是防 SQL 注入。
  - 认证、授权、上传、内容输出都有安全问题。

## 本周实践

- 选 1 个前端性能问题和 1 个后端安全问题做分析。
- 写一份“输入不可信”的清单。
- 用通俗话解释：为什么性能和安全都应该尽早纳入设计。

## 学完你应该能说清

- 什么是性能预算，为什么需要它。
- 为什么权限边界是安全设计核心。
- 为什么“先把功能做完再补安全”通常代价更高。
