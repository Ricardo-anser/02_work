# 第19周 文件、内容与缓存链路

## 本周你到底在学什么

这周学的是一个内容系统真正难的地方：不是“把文件传上去”，而是“数据怎么校验、内容怎么处理、缓存怎么失效”。

## 先把原理讲透

- 文件处理是一条链：接收、校验、存储、引用、回显。
- 内容处理是一条链：原始内容、解析、渲染、缓存、失效。
- 两条链最后会在“页面如何展示内容”上汇合。

## 最小例子

```js
import { createHash } from "node:crypto";

function md5(text) {
  return createHash("md5").update(text).digest("hex");
}
```

这个例子背后对应的是一个非常实用的原理：你要先确认“收到的内容是不是你以为的内容”。

## 放到 Vue3 / TS / Node.js 里

- Vue 页面关心的是内容和附件能否稳定展示。
- TS 可以约束上传响应、内容结构、缓存键模型。
- Node 负责摘要校验、内容转换、缓存更新和失效逻辑。

## 实际应用场景

- 文件上传。
- Markdown 内容渲染。
- 内容详情页缓存。

## 常见误区

- 误区：上传成功就结束了。
  - 后面还有回显、引用、删除、失效一整条链。
- 误区：缓存只要加上去就一定更快。
  - 缓存更新不及时，很容易把错误内容放大。

## 本周实践
