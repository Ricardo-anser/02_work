# 第17周 Vite 插件与构建

## 本周你到底在学什么

这周要拆穿“构建是黑盒”的错觉。你会知道源码从文件到产物，中间到底经过了哪些可编排步骤。

## 先把原理讲透

- 构建本质是“把适合开发者写的源码，变成适合运行环境消费的产物”。
- 插件系统本质是“在流水线关键节点插入自定义处理”。
- Vite 的价值不是神秘，而是把 dev 和 build 两套体验都做得足够可扩展。

## 最小例子

```ts
export default function bannerPlugin() {
  return {
    name: "banner-plugin",
    transform(code, id) {
      if (!id.endsWith(".ts")) return code;
      return `/* transformed */\n${code}`;
    },
  };
}
```

这个插件虽然简单，但已经体现了核心思想：源码不是固定不变的，它会经过构建期加工。

## 放到 Vue3 / TS / Node.js 里

- Vue SFC 编译离不开 Vite / Rollup 插件链。
- TS 文件在浏览器里不能直接跑，也要经过构建期处理。
- Node 工具脚本、虚拟模块、环境变量注入都可能利用插件能力。

## 实际应用场景

- 自动注入构建时间。
- 统一处理 Markdown 文件导入。
- 生成虚拟路由模块。

## 常见误区

- 误区：插件就是做花活。
  - 它经常直接服务项目组织和构建稳定性。
- 误区：dev 能跑就等于 build 没问题。
  - 两个阶段的处理链路并不完全相同。

## 本周实践

- 写一个最小 transform 插件。
- 观察同一份源码在开发期和构建期分别经过什么处理。
- 解释为什么“编译期能力”能帮你减轻运行时负担。

## 学完你应该能说清

- 构建和运行时的边界。
- 插件系统为什么是前端工程化的核心扩展点。
- 为什么理解构建流程会帮助你排查“明明本地好好的，打包就坏了”。
