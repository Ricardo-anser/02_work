# 第14周 事件循环、Stream 与 Buffer

## 本周你到底在学什么

这周开始真正进入 Node 运行时核心：任务怎么排队，数据怎么以二进制流动，为什么大文件处理不能像字符串那样随手来。

## 先把原理讲透

- 事件循环决定“什么时候轮到谁执行”。
- Buffer 解决的是“如何在内存里表示二进制数据”。
- Stream 解决的是“别把所有数据一次性吞进内存，而是边来边处理”。

## 最小例子

```js
console.log("start");

setTimeout(() => console.log("timer"), 0);
Promise.resolve().then(() => console.log("microtask"));

console.log("end");
```

这段代码最重要的不是输出，而是逼你建立一个顺序模型：为什么 `microtask` 会先于 `timer`。

## 放到 Vue3 / TS / Node.js 里

- Vue 页面里你经常感知不到事件循环，但 `nextTick`、批量更新离不开它。
- TypeScript 在这一周不是主角，它只是帮助你描述流数据接口。
- Node.js 则是主战场：上传、下载、日志、压缩、摘要计算都和 Stream/Buffer 强相关。

## 实际应用场景

- 文件上传。
- 大日志文件逐行处理。
- 图片、音视频、附件服务。

## 常见误区

- 误区：异步就等于多线程。
  - Node 大量异步能力来自事件循环，不等于业务代码同时多线程执行。
- 误区：读文件就是 `readFile()`。
  - 大文件和长连接场景更适合流式处理。
