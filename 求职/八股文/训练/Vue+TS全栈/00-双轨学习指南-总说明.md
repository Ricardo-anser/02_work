# Vue3 + TypeScript + Node.js 双轨 24 周学习指南

## 00. 使用说明

- 目标读者：已具备 HTML / CSS / ES6 基础，但尚未系统掌握 Vue3 + TypeScript + Node.js 全栈协作的学习者。
- 时间基线：`24` 周，按 `2 小时/天` 有效学习时长设计；其中 1 小时需保持连续编码或调试。
- 交付形态：两份结构对齐但内容正交的 Markdown 指南。
  - [底层原理掌握者](./01-底层原理掌握者-24周学习指南.md)
  - [应用交付实践者](./02-应用交付实践者-24周学习指南.md)
  - [底层原理掌握者-逐周详解目录](./04-底层原理掌握者-24周详解/README.md)
  - [应用交付实践者-逐周详解目录](./05-应用交付实践者-24周详解/README.md)
- 并行规则：两条路线共享同一周次与同一日历编号，便于同周对照与交叉补位，但不要求同一天同时执行两条路线。

### 学习输出约定

- 每周必须产出：
  - 1 个周度交付物
  - 1 次周度通关检测
  - 1 份错误清单或复盘摘要
- 每天必须记录：
  - 当日任务是否完成
  - 实际编码行数
  - 实际调试次数
  - 实际文档阅读量
  - 是否触发压缩版替代任务

## 01. 前置知识

### 必须掌握

- JavaScript 基础语法：`let/const`、函数、对象、数组、高阶函数、Promise。
- DOM 与浏览器基础：事件、表单、网络请求、模块化脚本加载。
- 命令行基础：会在终端中切换目录、安装依赖、运行脚本。
- 如果这部分不稳，先补 [03-JavaScript基础语法与DOM前置补充](./03-JavaScript基础语法与DOM前置补充.md)。

### 训练中补齐

- Git 基础：分支、提交、对比与回滚。
- SQL 基础：`SELECT`、`INSERT`、`UPDATE`、索引与事务直觉。
- Linux/部署基础：进程、环境变量、日志文件。

### 暂时忽略

- 商业 UI 框架封装细节。
- 复杂云原生编排。
- 多端应用与原生移动端集成。

## 02. 双轨对齐规则

### 共用知识基座

- 模块系统与加载语义
- 异步模型与错误边界
- 类型与接口抽象
- 状态、数据流与副作用控制
- 编译期与运行时分工
- 工程化、部署与观测

### 双轨差异

| 维度 | 底层原理掌握者 | 应用交付实践者 |
| --- | --- | --- |
| 主目标 | 解释机制、手写最小实现、复现实验 | 搭建可运行系统、完成联调、上线与观测 |
| 验收锚点 | 源码可读、实验可复现、概念可迁移 | 功能可交付、链路可验证、系统可部署 |
| 每日产出 | 实验脚本、原理笔记、最小实现 | 页面、接口、脚本、部署配置 |
| 失败回滚 | 缩成最小语义实验 | 缩成最小可用功能片段 |

## 03. 每日条目模板

后续若要把本蓝图扩展为逐日正式训练手册，每一天都必须展开为以下字段：

| 字段 | 说明 |
| --- | --- |
| 主题 | 当天唯一概念焦点，避免多个难点混杂。 |
| 任务 | 编码、调试、阅读或复盘任务，必须可执行。 |
| 验证代码/实验 | 可直接运行的最小验证动作。 |
| 破坏性测试 | 主动制造一个失败场景，验证理解边界。 |
| 来源依据 | 使用本文件中的证据索引 ID。 |
| 量化指标 | 代码行数、调试次数、阅读量三项至少两项有目标。 |
| 压缩版替代任务 | 当天未完成时的最低保真回滚方案。 |

## 04. 版本与证据口径

### 版本基线

- 绝对日期基线：`2026-05-25`
- Vue：以 `Vue 3.4+` 概念为主，遇到 `defineModel`、异步组件、Suspense 等语法糖时显式标版本。
- TypeScript：以 Handbook 当前结构为索引，涉及模板字面量类型时标注 `TS 4.1+`，涉及较新的语义再单独说明。
- Node.js：后端主线默认以 `Node.js 20 LTS` 的 ESM / CJS、Web Crypto、Streams 语义为准。

### 2024 Q2 生态健康度快照”固定模板

本指南不假设存在一份外部统一白皮书，而是要求对候选工具按同一口径采样：

| 指标 | 采样窗口 | 数据入口 | 用途 |
| --- | --- | --- | --- |
